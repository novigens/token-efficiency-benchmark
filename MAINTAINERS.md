# Maintainers

This file lists active project maintainers and contact channels.

## Active maintainers

- Ken Quah (Novigens Inc.), project lead

## Contact

- **General questions, bug reports, RFCs**: GitHub issues on this repository.
- **Code of Conduct concerns**: contact the project maintainers via a private channel; one will be added here once the public home is established.
- **Security disclosures**: please open a private issue or contact the maintainers directly.

## Decision-making

For most changes (bug fixes, new adapters, new templates), the regular pull-request review is sufficient. For substantive changes to the settled design properties (see `CONTRIBUTING.md`), an RFC is required. RFCs are tracked as `rfc`-tagged issues and require maintainer approval to merge.

## How to become a maintainer

The current criteria, in rough order of weight: sustained contribution of high-quality adapters or templates, demonstrated independence in review, ability to articulate the design's settled properties to new contributors, and willingness to spend the time. Maintainers are added by consensus of existing maintainers.
